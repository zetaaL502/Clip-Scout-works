*MASTER BUILD DIRECTIVE: CLIPSCOUT**

You are building a complete, fully functional web application called ClipScout. It is a B-roll video scouting tool for YouTube creators. The user pastes their script, the app chunks it into segments using Groq AI, fetches matching video clips from Pexels (via Supabase Edge Function proxy) and GIFs from Giphy (direct browser call) per segment, lets the user select multiple clips per segment, and downloads all selected clips as a single ZIP file.

Read this entire document before writing a single line of code. Follow every instruction exactly.

* Install and use client-zip npm library for ZIP export
* All app data stored in localStorage
* Dark mode only. Background: #0a0a0a. Cards: #111111 or #1a1a1a. Accent: #22c55e. Text: white and gray-400

**5. GROQ — SCRIPT CHUNKING**

Call Groq directly from the browser:
* Endpoint: `https://api.groq.com/openai/v1/chat/completions`
* Header: `Authorization: Bearer {clipscout_groq_key}`
* Model: `llama-3.3-70b-versatile`
* `response_format: { type: "json_object" }`
* Timeout: 15 seconds. If timeout: red toast "Groq timed out. Please try again." + re-enable button

Use exactly this prompt, inserting the full script where it says {{SCRIPT}}:

```
You are a video production assistant helping a YouTube creator scout B-roll footage for a long-form video.

Read the full script carefully. Split it into logical segments of approximately 50–75 words each. Aim for 20–35 segments total. Never cut mid-sentence. Never make a segment shorter than 30 words or longer than 100 words.

For each segment generate:
- "pexels_keywords": 3–5 specific visual words describing cinematic, landscape, nature, or action footage. Example: "busy city street night rain"
- "giphy_keywords": 2–3 words for a fun expressive GIF. Example: "mind blown"
- "duration_estimate": estimated speaking time e.g. "~15 seconds"

Return ONLY valid raw JSON with no markdown, no explanation, no code blocks:
{
  "segments": [
    {
      "order_index": 1,
      "text_body": "exact script text for this segment",
      "pexels_keywords": "keywords here",
      "giphy_keywords": "giphy search terms",
      "duration_estimate": "~15 seconds"
    }
  ]
}

Full script:
{{SCRIPT}}
```

After receiving the response:
* Parse the JSON
* Assign a unique local ID to each segment
* Save all segments to localStorage under `clipscout_segments`
* Navigate to Page 2

**6. VIDEO FETCHING FUNCTIONS**

**Pexels (via Supabase Edge Function — already deployed):**
* Call: POST `https://sauffvkpbpytzwojnopt.supabase.co/functions/v1/pexels-proxy`
* Headers: `{ "Content-Type": "application/json" }`
* Body: `{ "keywords": string, "page": number }`
* Returns array of: `{ id, thumbnail_url, media_url }`

**Giphy (direct browser call):**
* Endpoint: `https://api.giphy.com/v1/gifs/search`
* Params: `api_key={clipscout_giphy_key}&q={keywords}&limit=4&offset={(page-1)*4}&rating=g`
* Extract per result: `id`, `images.fixed_height_still.url` (thumbnail), `images.original.gif` (gif URL)
* Returns array of: `{ id, thumbnail_url, media_url }`

Both functions must return a consistent clip object shape:
```typescript
{
  id: string,
  segmentId: string,
  source: 'pexels' | 'giphy',
  thumbnail_url: string,
  media_url: string
}
```

**8. SEGMENT CARDS — CRITICAL, READ EVERY WORD**

Segment cards stack vertically, one per segment, full width. Each card is a completely self-contained unit with its own independent state.

Card structure:
```
┌──────────────────────────────────────────┐
│  SEGMENT 3 of 27              ~15 seconds │
│──────────────────────────────────────────│
│  "The script text for this segment goes  │
│   here and wraps naturally..."           │
│──────────────────────────────────────────│
│              [＋ Add 4 More ▾]           │
│──────────────────────────────────────────│
│  [clip]  [clip]                          │
│  [clip]  [clip]                          │
│  ... more clips appended below ...       │
└──────────────────────────────────────────┘
```

**Initial clip loading:**
* When Page 2 first renders, automatically fetch 4 Pexels clips for the FIRST 5 segments in parallel using each segment's `pexels_keywords`
* Use Intersection Observer API to detect when remaining segment cards enter the viewport and lazy-load their clips at that moment
* Never fetch all segments at once on page load
* While fetching show exactly 4 skeleton loader boxes: `animate-pulse bg-gray-800 rounded-lg` in the 2-column grid

**"＋ Add 4 More ▾" button — CRITICAL BEHAVIOR:**
* On click: show a small dropdown directly below the button with exactly two options: **"Pexels"** and **"Giphy"**
* Each segment independently tracks two page counters: `pexels_page` (starts at 1, already used for initial load) and `giphy_page` (starts at 0)
* User picks **Pexels**:
  - Increment THIS segment's `pexels_page` by 1
  - Call pexels-proxy with THIS segment's `pexels_keywords` and new page number
  - Append the 4 returned clips to the BOTTOM of THIS segment's clip grid only
  - No other segment is affected. No page refresh. No re-render of other segments.
* User picks **Giphy**:
  - Increment THIS segment's `giphy_page` by 1
  - Call Giphy directly with THIS segment's `giphy_keywords` and new page number
  - Append the 4 returned GIFs to the BOTTOM of THIS segment's clip grid only
  - No other segment is affected. No page refresh. No re-render of other segments.
* User picks **Pexels again**:
  - Increment `pexels_page` again, fetch next 4 Pexels clips, append below existing
* User picks **Giphy again**:
  - Increment `giphy_page` again, fetch next 4 Giphy GIFs, append below existing
* While loading new clips: show 4 skeleton loaders at the bottom of ONLY that segment's grid
* After each "Add 4 More" click: disable the button for 3 seconds, show countdown "Wait 2s…"
* The rest of the page must NOT reload, refresh, or re-render. Only that segment's clip grid updates.

**Clip grid layout:**
* 2 columns on mobile (<640px)
* 4 columns on desktop (>1024px)
* Each clip cell: thumbnail image, 16:9 aspect ratio, object-cover, loading="lazy"
* On hover: show semi-transparent dark overlay with white play icon centered
* Min touch target: 44x44px

**Clip selection:**
* Multiple clips can be selected per segment
* Clicking an unselected clip: adds green border (border-4 border-green-500 rounded-lg) + green checkmark badge top-right corner. Adds clip ID to `clipscout_selections` in localStorage.
* Clicking a selected clip: removes green border and checkmark. Removes clip ID from `clipscout_selections` in localStorage.
* Selection state persists in localStorage and survives page refresh

**9. EXPORT ZIP — CLIENT SIDE**

When user clicks Export ZIP:
1. Read all selected clip IDs from `clipscout_selections` in localStorage
2. Match each ID to its clip object from `clipscout_clips`
3. Sort clips by their segment's `order_index`
4. Show a full-screen dark overlay with:
   - Text: "Preparing your clips…"
   - Progress bar: "Downloading X of Y"
5. For each selected clip fetch the `media_url` directly using browser fetch() to get the blob
6. Generate a `script.txt` file from the full script text
7. Use client-zip to bundle everything:
   - Video files named: `01_scene.mp4`, `02_scene.mp4` (zero-padded)
   - GIF files named: `03_scene.gif` etc.
   - `script.txt` in the root of the zip
8. Trigger browser download of `clipscout_export.zip`
9. On any single clip download error: red toast "Clip [N] failed, skipping" — continue downloading the rest
10. Dismiss overlay when complete
11. Show green toast: "Export ready! Downloading now…"
